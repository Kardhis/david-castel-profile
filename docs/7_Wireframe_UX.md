# Wireframe y UX

## Objetivo

Diseñar la estructura visual inicial.

---

# Diseño

- Header.
- Hero.
- Secciones.
- Footer.
- Navegación.
- Responsive móvil.

---

# Herramientas posibles

- Figma.
- Excalidraw.
- Canva.
- Boceto HTML.

---

# Objetivo principal

Conseguir que el visitante:

- entienda rápidamente quién eres,
- perciba profesionalidad,
- valide tu capacidad técnica,
- explore tus proyectos,
- y quiera contactarte.

---

# 6.1 — Filosofía UX recomendada

Tu portfolio debe sentirse como:

> un producto digital moderno

NO:

> un CV online tradicional.

## La UX debe transmitir

- fluidez,
- claridad,
- velocidad,
- simplicidad elegante,
- profesionalidad,
- modernidad,
- sensación premium.

---

# 6.2 — Principio estratégico principal

La navegación debe ser:

> extremadamente simple.

Porque tu objetivo NO es:

> “hacer una web compleja”.

Tu objetivo es:

> transmitir capacidad profesional rápidamente.

---

# 6.3 — Tipo de estructura recomendada

## Single Page Application (SPA)

Con navegación smooth scroll.

### ¿Por qué?

Porque transmite:

- modernidad,
- fluidez,
- experiencia premium,
- sensación SaaS moderna.

## Recomendación UX principal

Navbar fija minimalista con navegación suave entre secciones.

### Secciones recomendadas

- Hero
- Sobre mí
- Stack
- Proyectos
- Filosofía de trabajo
- Experiencia
- Contacto
- Footer

---

---

# 6.5 — HERO SECTION

La sección más importante.

## Objetivo UX

Generar impacto inmediato.

## Layout recomendado

### Izquierda

Contenido principal.

### Derecha

Elemento visual potente.

Por ejemplo:

- mockup,
- UI flotante,
- terminal elegante,
- cards tecnológicas,
- visual SaaS.

## Contenido recomendado

### Headline potente

> Transformo ideas en aplicaciones web y SaaS reales utilizando arquitectura moderna, automatización e inteligencia artificial.

### Subheadline

### CTAs

- Ver proyectos
- Contactar

### Indicadores técnicos

- Full Stack Engineering
- AI-Assisted Development
- SaaS Architecture
- Production Deployments

## UX importante

- Mucho espacio visual
- Jerarquía clara
- Animaciones suaves
- Lectura inmediata

## Objetivo final

Que el usuario piense en menos de 5 segundos:

> “Este desarrollador parece moderno y profesional.”

---

# 6.6 — ABOUT SECTION

## Objetivo UX

Humanizar y reforzar posicionamiento.

## Layout recomendado

Texto + visual profesional.

## Contenido recomendado

- Quién eres
- Qué construyes
- Cómo trabajas
- IA aplicada profesionalmente
- Filosofía técnica

## UX recomendada

- Texto corto y claro
- Mucho espacio visual
- Lectura cómoda

## Evitar

❌ biografía larga

❌ párrafos enormes

---

# 6.7 — STACK SECTION

## Objetivo UX

Validación técnica rápida.

## Layout recomendado

Grid moderna de tecnologías.

### Categorías recomendadas

- Frontend
- Backend
- Infraestructura
- Bases de datos
- IA / Automatización

## UX recomendada

- Badges modernos
- Hover elegante
- Organización limpia
- Visual tecnológico

---

# 6.8 — PROJECTS SECTION

Probablemente la segunda sección más importante.

## Objetivo UX

Demostrar capacidad real.

## Layout recomendado

Cards premium grandes.

## Cada proyecto debe mostrar

- Captura premium
- Nombre
- Problema resuelto
- Stack
- Arquitectura
- Estado
- CTA
  - Ver proyecto
  - GitHub

## UX recomendada

- Hover elegante
- Motion suave
- Layout limpio
- Sensación SaaS real

## MUY IMPORTANTE

Pocos proyectos.

Pero:

> extremadamente bien presentados.

# 6.14 — Principios UX clave

- Claridad inmediata
- Jerarquía visual fuerte
- Mucho espacio visual
- Navegación simple
- Scroll natural
- Motion elegante
- Responsive premium
- Carga rápida

---

# 6.15 — Qué NO debemos hacer

❌ demasiadas páginas

❌ navegación compleja

❌ texto excesivo

❌ demasiadas animaciones

❌ demasiados efectos

❌ layouts sobrecargados

❌ diseño tipo dashboard antiguo

❌ portfolio tipo CV PDF
