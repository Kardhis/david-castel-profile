# Identidad visual

## Objetivo

Definir el estilo visual del portfolio.

---

# Elementos a definir

- Paleta de colores.
- Tipografías.
- Modo oscuro.
- Estilo moderno y profesional.
- Animaciones.
- Cards.
- Botones.
- Espaciados.

---

# Resultado esperado

Guía visual del proyecto.

---

# Objetivo visual principal

Tu identidad visual debe transmitir:

- ingeniería moderna,
- producto digital,
- profesionalidad,
- modernidad tecnológica,
- sofisticación,
- capacidad real de ejecución.

## Objetivo estratégico

Cuando alguien entre en la web debe pensar automáticamente:

> “Esto parece el producto de un ingeniero moderno especializado en SaaS y tecnología.”

---

# 5.1 — Concepto visual principal

## Modern Tech Premium

La identidad visual debe sentirse:

- tecnológica,
- elegante,
- premium,
- minimalista,
- moderna,
- profesional,
- limpia,
- sofisticada.

---

# 5.2 — Dirección visual recomendada

La identidad visual debe inspirarse en:

Minimalismo premium moderno.

## Vercel

Ingeniería visual limpia y tecnológica.

## Stripe

Sensación premium y layouts sofisticados.

## Raycast

Dark mode elegante y refinado.

## Supabase

Tecnología + producto + SaaS moderno.

---

# 5.3 — Paleta de colores recomendada

## Base principal

Dark mode premium.

## Fondo principal

NO negro absoluto.

Mejor tonos tipo:

- graphite,
- midnight,
- charcoal,
- dark navy.

## Sensación deseada

Oscuro elegante.

No “modo hacker”.

## Colores de acento recomendados

- Azul tecnológico elegante.
- Violeta moderno suave.
- Cyan sofisticado.
- Verde SaaS moderno.

## Importante

Los acentos deben:

- resaltar,
- pero NO dominar.

## Estilo recomendado

Contraste elegante y controlado.

---

# 5.5 — Estilo de componentes

Los componentes deben sentirse:

- modernos,
- limpios,
- elegantes,
- tecnológicos,
- refinados.

## Componentes recomendados

### Cards premium

- bordes suaves,
- sombras ligeras,
- dark surfaces,

### Botones modernos

- minimalistas,
- sólidos,

### Inputs refinados

### Badges tecnológicos

### Secciones muy respiradas

## Muy importante

Mucho espacio visual.

---

# 5.6 — Estilo de layout

La estructura debe sentirse:

- limpia,
- organizada,
- moderna,
- fluida,
- elegante.

## Recomendaciones

- Contenedores amplios.
- Márgenes generosos.
- Grid moderno.
- Ritmo visual equilibrado.
- Jerarquía tipográfica clara.

## Evitar

❌ sensación compacta

❌ demasiado contenido visible a la vez

❌ densidad visual excesiva

---

# 5.7 — Estilo de animaciones

Las animaciones deben transmitir:

- sofisticación,
- fluidez,
- modernidad,
- refinamiento.

## Recomendaciones

- Scroll reveal suaves.
- Hover animations discretas.
- Microinteracciones elegantes.
- Motion minimalista.
- Transiciones fluidas.

## Importante

Las animaciones deben acompañar.

NO:

❌ llamar más la atención que el contenido.

---

# 5.8 — Estilo de iconografía

- Minimalista.
- Moderna.
- Tecnológica.
- Coherente.

## Recomendaciones

- Lucide Icons
- Heroicons

## Evitar

❌ iconos caricaturescos

❌ exceso de iconos

❌ iconografía demasiado colorida

---

# 5.9 — Estilo visual de proyectos

Los proyectos deben verse como:

> productos reales.

## Recomendaciones

- Mockups elegantes.
- Capturas premium.
- Tarjetas estilo SaaS.
- Tecnologías visibles.
- Estado del proyecto visible.
- CTA claros.

## Sensación deseada

> “Esto parece un producto real en producción.”
